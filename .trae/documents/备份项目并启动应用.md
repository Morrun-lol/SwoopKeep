# Windows语音记账应用 - 备份与启动计划

## 📦 第一步：项目备份

**备份目标**：将整个项目目录复制为备份版本，防止后续修改导致项目损坏

**备份策略**：
- 备份路径：`d:\Trea 项目_backup_[日期时间]`
- 备份内容：整个项目目录（排除 node_modules、out、.vite 等构建产物）
- 备份方式：使用 PowerShell 的 `Copy-Item` 命令递归复制

**执行命令**：
```powershell
$timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
$backupPath = "d:\Trea 项目_backup_$timestamp"
Copy-Item -Path "d:\Trea 项目" -Destination $backupPath -Recurse -Force -Exclude @("node_modules", "out", "dist", ".vite")
```

## 🚀 第二步：启动应用测试

**启动目标**：运行开发模式，验证应用功能

**执行步骤**：

1. **检查依赖**
   - 确认 node_modules 是否存在
   - 如不存在则运行 `npm install`

2. **启动开发服务器**
   - 命令：`npm run dev`
   - 这将启动 Electron + Vite 开发环境

3. **验证启动**
   - 检查终端输出是否有错误
   - 确认应用窗口正常打开（1200x800）
   - 验证数据库初始化成功

4. **功能测试建议**（启动后手动测试）
   - ✅ 测试语音记账页面的"模拟测试"功能
   - ✅ 检查统计图表是否正常加载
   - ✅ 验证历史记录查询功能
   - ✅ 测试预算配置页面
   - ✅ 检查数据导入功能

---

## ⚠️ 注意事项

1. **备份期间**：
   - 确保有足够的磁盘空间（约 200-500MB）
   - 备份过程可能需要 1-3 分钟

2. **启动期间**：
   - 首次启动会请求麦克风权限（需允许）
   - DeepSeek API Key 已配置（.env文件中）
   - 科大讯飞语音识别 API Key 未配置，语音识别功能会被禁用
   - 数据库文件会自动创建在用户数据目录

3. **如遇错误**：
   - 端口占用：关闭其他 Electron 应用
   - 依赖缺失：运行 `npm install`
   - 数据库错误：删除用户数据目录下的 expenses.db 重新初始化
